Tăng Tốc Website: Core Web Vitals & PageSpeed Insights

Tốc độ website ảnh hưởng trực tiếp đến SEO ranking và trải nghiệm người dùng. Google xác nhận Core Web Vitals là yếu tố xếp hạng. Bài viết này hướng dẫn bạn đạt 90+ điểm PageSpeed.

Core Web Vitals là gì?

Core Web Vitals là bộ 3 chỉ số Google dùng để đo trải nghiệm người dùng thực tế:

  • LCP (Largest Contentful Paint): Thời gian render phần tử lớn nhất — mục tiêu < 2.5 giây
  • FID (First Input Delay): Thời gian phản hồi tương tác đầu tiên — mục tiêu < 100ms
  • CLS (Cumulative Layout Shift): Độ ổn định layout — mục tiêu < 0.1

1. Tối ưu hình ảnh

Hình ảnh thường chiếm 50-80% dung lượng trang. Đây là bước đầu tiên và tác động lớn nhất.

  • Chuyển sang format WebP — giảm 25-35% so với JPEG
  • Dùng loading="lazy" cho hình ngoài viewport
  • Đặt width và height cố định để tránh layout shift
  • Dùng <picture> với srcset cho responsive

2. Minify CSS, JS, HTML

Loại bỏ khoảng trắng, comment, code thừa. Sử dụng build tools:

  • CSS: cssnano, PurgeCSS
  • JS: Terser, UglifyJS
  • HTML: html-minifier
💡 Mẹo: Dùng PurgeCSS để loại bỏ CSS không sử dụng — có thể giảm 80-90% file CSS.

3. Defer & Async JavaScript

JavaScript blocking là nguyên nhân chính làm chậm FID. Thêm defer hoặc async cho tất cả script không critical.

4. Preload tài nguyên quan trọng

5. CDN — Gần người dùng hơn

Content Delivery Network phân phối nội dung từ server gần nhất. Giảm latency 50-80%. Các CDN miễn phí: Cloudflare, jsDelivr.

6. Browser Caching

Đặt Cache-Control headers hợp lý. CSS/JS nên cache lâu (1 năm) với hash trong filename. HTML cache ngắn hoặc no-cache.

7. Giảm HTTP Requests

  • Gộp CSS files thành 1
  • Inline critical CSS
  • Dùng SVG sprites thay nhiều icon files
  • Dùng CSS thay hình ảnh cho gradient, shadow

Kiểm tra và theo dõi

Sử dụng các công cụ miễn phí:

  • Google PageSpeed Insights — điểm tổng và gợi ý
  • GTmetrix — waterfall chart chi tiết
  • WebPageTest — test từ nhiều location
  • Google Search Console — Core Web Vitals thực tế
